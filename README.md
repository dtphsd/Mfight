# 📦 Шаблоны документации для нового проекта

## Что это?

Набор из 5 готовых шаблонов документации для **любого** веб-проекта (React, Vue, Next.js, и т.д.).

Положи эти файлы в корень своего проекта → попроси AI-агента заполнить → получи полную документацию.

---

## 📋 Файлы в наборе

| Файл | Зачем |
|------|-------|
| `RULES.md` | **Начни с этого!** Правила: когда и как обновлять остальные доки. Деплой. Чеклист. |
| `PROJECT-INFO.md` | Полное описание проекта: фичи, стек, дизайн-система, адаптивность |
| `PROJECT_STRUCTURE.md` | Структура папок и файлов с описанием каждого элемента |
| `SECURITY_ARCHITECTURE.md` | Безопасность: CORS, rate limiting, auth, шифрование, GDPR |
| `DATABASE_ARCHITECTURE.md` | Схема БД, таблицы, миграции, правила изменений |

---

## 🚀 Как использовать

### Шаг 1: Скопируй
Скопируй все 5 `.md` файлов в **корень** своего нового проекта.

### Шаг 2: Попроси AI заполнить
Напиши своему AI-агенту (Cursor, Windsurf, Gemini, Claude и т.д.):

> **"Изучи весь проект и заполни шаблоны документации: RULES.md, PROJECT-INFO.md, PROJECT_STRUCTURE.md, SECURITY_ARCHITECTURE.md, DATABASE_ARCHITECTURE.md. Замени все {{плейсхолдеры}} реальными данными и следуй инструкциям в <!-- AI-АГЕНТ --> комментариях. Удали все комментарии после заполнения."**

### Шаг 3: Поддерживай актуальность
После каждого крупного изменения в проекте говори агенту:

> **"Обнови документацию по правилам из RULES.md"**

---

## 💡 Советы

- **`{{Плейсхолдеры}}`** — замени на реальные данные проекта
- **`<!-- AI-АГЕНТ: ... -->`** — инструкции для AI, удалить после заполнения
- Если проекту не нужен бэкенд — удали секции Backend и DATABASE_ARCHITECTURE.md
- Если нет базы данных — удали DATABASE_ARCHITECTURE.md
